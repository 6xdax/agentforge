## 一、核心定义与问题建模

意图识别，本质上是一个**文本分类**问题，是对话系统的“方向盘”。它负责将用户的自然语言输入映射到预定义的意图类别上。

**数学表达**：
给定意图集合 \( C = \{c_1, c_2, ..., c_n\} \) 和用户输入文本 \( x \)，目标是找到一个函数 \( f \)，使得预测意图 \( \hat{y} \) 满足：

\[ \hat{y} = \underset{c \in C}{\operatorname{argmax}} \ P(c|x) \]

**两大应用场景**：
- **单轮意图识别**：仅根据当前用户单次输入进行判断，不考虑历史对话。
- **多轮意图识别**：需结合历史对话上下文，对用户在当前轮次的意图进行判断，处理意图的继承、切换和关联。

## 二、方法体系分类

根据应用场景和技术路线，意图识别的实现方法可系统性地划分为以下几类：

### 1. 单轮意图识别方法

| 方法大类 | 具体方案 | 核心原理 | 特点与适用场景 |
| :--- | :--- | :--- | :--- |
| **规则匹配** | - **关键词/正则匹配** <br>- **分词器 + 规则库** (如jieba, HanLP) | 对文本分词后，匹配预置的关键词、短语或正则表达式，命中即返回对应意图。 | **优点**：速度快，实现简单，冷启动成本低。<br>**缺点**：泛化能力极差，维护成本高。适用于意图明确、句式极其固定的场景。 |
| **向量检索** | **Embedding + 向量检索** (如 BGE, SimCSE模型 + Faiss, Milvus检索引擎) | 将用户输入和标准意图示例库中的句子都转换为语义向量，通过计算余弦相似度进行最近邻检索，返回Top-K相似意图。 | **优点**：具备一定泛化能力，无需繁琐规则，可实现零样本分类。<br>**缺点**：效果依赖Embedding模型质量和示例库覆盖度。适用于意图边界模糊、长尾意图多的场景。 |
| **传统机器学习** | **TF-IDF/N-gram + SVM/Naive Bayes** | 从文本中提取TF-IDF或N-gram统计特征，送入分类器进行训练和预测。 | **优点**：开始具备统计规律学习能力。<br>**缺点**：无法理解词序和深层语义上下文。已逐渐被深度学习方法取代。 |
| **深度学习(小模型)** | **BERT/DistilBERT + 全连接层** | 利用预训练模型提取深层双向语义特征，取 `[CLS]` 输出向量，通过全连接层和Softmax进行分类。 | **优点**：语义理解能力强，准确率高，是当前工业界高并发、低延迟场景的性价比之王。<br>**缺点**：需要标注数据，有训练和部署成本。适用于意图固定、有标注数据积累的核心业务场景。 |
| **大模型(LLM)** | **1. Prompt Engineering** (Zero/Few-Shot) | 在Prompt中描述所有意图，直接让LLM进行选择并输出。 | **优点**：无需训练，灵活度极高，支持零样本学习。<br>**缺点**：依赖提示词质量，大模型响应延迟、成本较高，输出格式可能不稳定。适用于冷启动、意图变更频繁的长尾场景。 |
| | **2. LLM微调 (SFT)** | 使用领域标注数据，对开源LLaMA、ChatGLM等大模型进行监督式微调。 | **优点**：可达到极高准确率，同时保留LLM的泛化与推理能力。<br>**缺点**：需要高质量标注数据和算力资源。适用于对准确率有极致要求的垂直领域。 |
| | **3. Function Calling (工具调用)** | 将意图定义为工具（Function），LLM根据用户输入自动判断意图并提取槽位参数。 | **优点**：同时完成意图识别和槽位填充，是构建AI Agent的核心范式。<br>**缺点**：依赖模型Function Calling能力。适用于需要调用外部API的复杂任务型对话。 |
| **融合方案** | **串并行混合框架** | 组合多种方法，按优先级排序。例如：**规则(高优先级) → 小模型(BERT) → 向量检索/大模型(兜底)**。 | **优点**：兼顾速度、成本、准确率和覆盖率，是工业级落地的最佳实践。 |

### 2. 多轮意图识别方法

多轮意图识别的核心挑战在于如何判断当前查询与历史对话的**上下文关联**。主要分为两大技术路线：

| 技术路线 | 具体方案 | 核心原理 | 特点 |
| :--- | :--- | :--- | :--- |
| **Pipeline (流水线式)** | **1. 基于上下文语义分割** | 利用深度学习模型（如基于NER框架）或微调大模型，判断当前查询是否应与历史上下文拼接，形成一个完整的语义片段再进行意图分类。 | 结构化，过程可控。但串联架构可能存在误差累积。 |
| | **2. 基于上下文槽位关联** | 通过规则或模型判断当前查询中的槽位（如地点、时间）是否继承自历史对话中缺失的槽位，从而关联意图。 | 在特定场景下有效，但仅靠槽位关联可能造成误判（例如历史提了地点“北京”，但当前新意图可能与北京无关）。 |
| **端到端 (End-to-End)** | **大模型端到端方案** | 直接将格式化后的全部历史对话和当前查询作为输入，由LLM统一理解上下文并输出当前意图。 | **优点**：模型统一，保留完整对话信息，能处理复杂的意图切换和继承。AI Agent的标准做法。<br>**缺点**：对基座模型能力要求高，输入长度增加，推理成本和延迟更高。需要构建包含多轮历史的指令微调数据集。 |

## 三、技术演进与未来挑战

**演进脉络总结**：
从**规则匹配**到**统计学习**，再到**深度学习(BERT)微调**，如今正迈向 **LLM驱动的智能路由与Agent** 时代。范式从“模型训练”转向“能力调用”，关注点也从单一分类准确率转向整个对话任务的完成率。

**未来核心挑战**：
1.  **意图模糊与多意图**：用户输入“这一单不要辣，顺便查下上单退款没”，需要精准识别并拆解为多个并行意图。
2.  **OOD拒识**：当用户问题超出系统能力范围时，模型需要有“自知之明”，主动拒识引导或拒答，而非强行分类到一个错误意图。
3.  **大小模型协同**：平衡大模型的理解与生成能力和微调小模型的速度与成本优势，实现在线推理的最佳性价比。
